Tutorial Operativo: JURIS-GESTIÓN-PRO

Aprende a dominar el sistema, gestionar expedientes y utilizar el asistente de JURIS-AI con el siguiente video explicativo.

Diseñando JURIS-GESTIÓN-PRO — Video de proceso y funcionalidades del sistema.

📘 Manual de Usuario — JURIS-GESTIÓN-PRO v2.0

Esta guía explica cada pantalla, botón y campo del sistema. Úsala como referencia rápida mientras trabajas. La aplicación se divide en dos zonas permanentes: el Menú Lateral izquierdo (navegación, fondo neutro integrado) y el Área de Trabajo central (pantallas del sistema). El Panel JURIS-AI es un overlay Copilot que se abre pulsando el botón ⚖️ JURIS-AI en la barra superior — no ocupa espacio permanente.

🗂️ 1. Menú Lateral y Barra Superior

La barra izquierda siempre está visible con fondo neutro claro (Windows 11 Fluent Design). El ítem activo se resalta con color azul y un indicador vertical. Haz clic en el logo para colapsar/expandir el menú.

  • 📊 Dashboard — Vista general con estadísticas, gráficos de barras y metas.
  • 👥 Clientes (CRM) — Directorio completo de todos los clientes registrados.
  • 📁 Expedientes — Gestión de todos los casos. Pantalla principal del sistema.
  • 📜 Protocolo — Libro digital de protocolo notarial.
  • 📅 Calendario — Cronograma de audiencias y plazos procesales.
  • ⚙️ Configuración — Ajustes del despacho, seguridad, tema visual y respaldos.
  • 📺 Tutorial (Ayuda) — Esta pantalla con el video y el manual.

Barra Superior

Muestra el título del módulo activo (centro) y el botón de acceso a JURIS-AI (derecha).

  • ⚖️ JURIS-AI — Abre el panel del asistente como overlay desde la derecha (estilo Copilot). No ocupa espacio permanente en la interfaz. Pulsa de nuevo o el botón ✕ para cerrarlo.
💡 Al hacer clic en cualquier ícono del menú, el área de trabajo cambia de pantalla instantáneamente sin recargar la aplicación.

📊 2. Dashboard Analítico

Pantalla de resumen ejecutivo del despacho. Se actualiza automáticamente cada vez que se realizan cambios en expedientes o audiencias.

Tarjetas de Estadística

  • Casos Activos — Total de expedientes con estado "Activo".
  • En Proceso — Total de expedientes con estado "En Proceso".
  • Casos Cerrados — Expedientes finalizados o archivados.
  • Honorarios Proyectados (L.) — Suma de todas las cuantías/honorarios registrados en audiencias y expedientes.

Gráfico de Distribución

Gráfico de torta (Chart.js) que muestra visualmente la proporción de casos por estado. Se actualiza automáticamente.

Estado del Protocolo

Indica si el libro notarial está dentro del límite de 200 folios. Muestra cuántos instrumentos hay registrados y el último folio utilizado.

Metas Mensuales

Barra de progreso que compara los honorarios acumulados contra la meta mensual configurada.

  • ✏️ Editar — Abre un cuadro para modificar la meta mensual de ingresos en Lempiras.

Botones de Exportación

📊 Exportar CSV

Descarga un archivo .csv con todos los expedientes y sus datos para abrir en Excel u otra hoja de cálculo.

📄 Reporte PDF

Genera e imprime un reporte ejecutivo completo del estado del despacho en formato PDF a través del navegador de impresión del sistema.

👥 3. Clientes — Directorio CRM

Aquí se registra y administra la información de todas las personas naturales o jurídicas que son clientes del despacho. Cada expediente debe estar vinculado a un cliente registrado.

Tabla de Clientes

Muestra columnas: DNI / RTN, Nombre Completo, Teléfono, Fecha Registro y botones de acción.

  • ✏️ Editar — Abre el formulario de edición del cliente (requiere PIN si está configurado).
  • 🗑️ Eliminar — Elimina el registro del cliente (requiere PIN). Esta acción no se puede deshacer.

Botón: + Nuevo Cliente

Abre el formulario de registro. Campos disponibles:

  • Nombre Completo * — Nombre o razón social. Campo obligatorio.
  • DNI / RTN * — Documento Nacional de Identidad o Registro Tributario Nacional. Obligatorio y único (no pueden existir dos clientes con el mismo DNI).
  • Teléfono — Número de contacto. Opcional.
  • Correo Electrónico — Email del cliente. Opcional.
  • Dirección — Ciudad y departamento. Opcional.
💡 Los campos marcados con * son obligatorios. Si se intenta registrar un DNI ya existente, el sistema mostrará un error de duplicado.
  • ✅ Guardar Cliente — Guarda el registro en la base de datos.
  • Cancelar — Cierra el formulario sin guardar.

📁 4. Expedientes y Casos — Pantalla Principal

Es el núcleo del sistema. Aquí se crean, consultan, gestionan y redactan todos los expedientes judiciales del despacho.

4.1 Barra Superior de Herramientas

  • 🔍 Buscar por cliente, ID, juzgado, materia...
    Campo de búsqueda en tiempo real. Al escribir cualquier texto, la tabla se filtra instantáneamente mostrando solo los expedientes que coincidan en cualquiera de sus columnas (ID del caso, nombre del cliente, juzgado, materia o estado). No requiere presionar Enter.
  • + Nuevo Expediente
    Abre el formulario para registrar un nuevo caso. Ver sección 4.3.
  • 📄 Redactar Escrito
    Abre el editor de documentos. Si ya hay un expediente seleccionado en la tabla, abre el editor directamente con ese caso. Si no hay ninguno seleccionado, muestra primero un selector para elegir el expediente al que pertenecerá el documento.

4.2 Pestañas de Estado

Filtran la tabla según el estado de los expedientes. Cada pestaña muestra un contador (badge) con la cantidad de casos en ese estado.

  • Activos — Casos recién abiertos o en etapa inicial.
  • En Proceso — Casos con actuaciones en marcha (audiencias, diligencias).
  • Cerrados — Casos finalizados, archivados o resueltos.

4.3 Registrar Nuevo Expediente

Formulario con los siguientes campos:

  • Materia Legal * — Selector desplegable. Al elegir la materia, el ID del expediente se genera automáticamente. Opciones: Civil, Penal, Laboral, Familia, Mercantil y Empresarial, Administrativo, Agrario, Constitucional, Notarial.
  • ID del Expediente (Auto-generado) 🔒 — Campo de solo lectura. El sistema genera el ID automáticamente con el formato AÑO-COD-NÚM (ejemplo: 2026-CIV-003). El número es secuencial por materia. No se puede editar manualmente.
  • Cliente * — Selector desplegable con todos los clientes registrados en el CRM. Muestra nombre y DNI para identificación segura. Obligatorio.
  • Juzgado — Nombre del juzgado o tribunal competente. Ejemplo: Juzgado de Letras Civil de Tegucigalpa. Opcional.
  • Cuantía / Honorarios Pactados (L.) — Monto en Lempiras de los honorarios acordados. Se suma a los "Honorarios Proyectados" del Dashboard. Opcional.
  • ✅ Guardar Expediente — Registra el expediente en la base de datos y lo muestra en la tabla.
  • Cancelar — Cierra el formulario sin guardar.

4.4 Panel de Gestión (clic sobre una fila)

Al hacer clic sobre cualquier fila de la tabla, se abre el panel de gestión de ese expediente. Muestra el ID del caso y el nombre del cliente en la cabecera.

Cambiar Estado del Expediente:

🟢 Activo

Mueve el expediente a la pestaña "Activos". El botón del estado actual aparece resaltado.

🟡 En Proceso

Mueve el expediente a la pestaña "En Proceso".

🔴 Cerrado

Archiva el expediente en la pestaña "Cerrados". El sistema muestra confirmación antes de aplicar.

💡 El botón del estado actual aparece resaltado con borde más prominente. No puedes cambiar un expediente al estado en que ya se encuentra.

Botones de Acción:

  • 🗑️ Eliminar — Elimina permanentemente el expediente. Requiere PIN de administrador si está configurado. Esta acción no se puede deshacer.
  • ✏️ Editar — Abre el formulario de edición para modificar la materia legal y el juzgado del expediente. Requiere PIN.
  • 📄 Redactar Documento — Abre el editor de escritos con este expediente como contexto activo.

4.5 Editor de Documentos y Plantillas

Panel de redacción jurídica que se despliega en la parte inferior de la pantalla de Expedientes. Vincula el documento a un expediente específico e inyecta automáticamente los datos del cliente y caso.

  • 📄 Caso: [ID] — Muestra el ID del expediente activo al que se le está redactando el documento.
  • 🔍 Escribe para buscar (ej: divorcio, penal)...
    Buscador de plantillas en tiempo real. Al escribir una palabra clave, filtra el selector desplegable mostrando solo las plantillas que coincidan. Si hay una sola coincidencia, la selecciona automáticamente.
  • -- Selecciona una Plantilla -- (selector desplegable)
    Lista todas las plantillas jurídicas disponibles organizadas en categorías: Derecho Civil, Derecho Familiar, Derecho Laboral, Derecho Mercantil y Empresarial, Derecho Penal y Administrativo, Otros Formatos Comunes. Más de 50 plantillas precargadas.
  • Cargar y Autocompletar
    Carga la plantilla seleccionada en el área de edición e inyecta automáticamente los datos del caso: nombre del cliente, DNI, teléfono, domicilio y juzgado en los campos correspondientes de la plantilla.
  • Área de edición (textarea) — Campo de texto libre donde se muestra el contenido de la plantilla ya autocompletado. Puedes editar, agregar o quitar texto libremente antes de exportar. Soporta formato Markdown básico: # Título, **negrita**, - ítem de lista.
📄 Exportar como Word (.docx)

Convierte el texto del editor a un documento Word profesional con: papel tamaño Legal (8.5"×14"), fuente Times New Roman 14pt, espaciado 1.5, márgenes estándar y alineación justificada. Abre el explorador de archivos del sistema para elegir dónde guardar el archivo.

Cerrar

Cierra el editor de documentos y vuelve a la vista normal de la tabla de expedientes.

⚠️ El editor no guarda automáticamente. Si cierras sin exportar, el contenido se perderá. Exporta a Word antes de cerrar.

4.6 Selector de Caso (al usar "Redactar Escrito" sin expediente activo)

Si pulsas 📄 Redactar Escrito desde la barra superior sin haber hecho clic en ninguna fila primero, aparece este modal con la lista completa de todos los expedientes (activos, en proceso y cerrados).

  • 🔍 Filtrar expedientes... — Buscador para localizar rápidamente el caso deseado dentro del modal.
  • Cada expediente en la lista muestra su ID, nombre del cliente, materia y estado.
  • Al hacer clic sobre uno de ellos, se cierra el modal y se abre el editor ya vinculado a ese caso.

📜 5. Protocolo Notarial

Módulo exclusivo para abogados con función notarial. Lleva el control digital del libro de protocolo, instrumentos autorizados y control de folios utilizados, con alerta cuando se acerca al límite de 200 folios por tomo.

Tarjetas de Resumen

  • Instrumentos Registrados — Cantidad total de escrituras e instrumentos en el tomo actual.
  • Último Folio Utilizado — Número del último folio registrado. El sistema alerta cuando supera los 180 folios.

Tabla de Protocolo

Columnas: No. (número correlativo), Fecha, Cliente / Otorgante, Naturaleza del Acto, Folios y botones de acción.

  • 👁️ Ver Ficha — Muestra el detalle completo del instrumento.
  • 🗑️ Eliminar — Elimina el instrumento (requiere PIN).

Botón: + Nuevo Instrumento

Abre el formulario de autorización de nuevo instrumento notarial. Campos:

  • Cliente / Otorgante (ID) — Número de ID del cliente en el CRM. El sistema vincula el instrumento al cliente registrado.
  • Naturaleza del Acto — Tipo de escritura o instrumento. Ejemplo: Escritura de Compraventa, Poder General.
  • Cantidad de Folios a Utilizar — Número de folios que ocupará el instrumento. El sistema suma automáticamente al total del tomo.
  • Autorizar Registro — Guarda el instrumento en el protocolo y actualiza el contador de folios.

Botón: 📄 Exportar Índice

Genera el índice notarial completo del tomo (lista numerada de todos los instrumentos con fecha, otorgante y naturaleza del acto) y abre el diálogo de guardado para exportarlo como archivo .md.

💡 El Dashboard muestra en tiempo real el estado del protocolo. Cuando el conteo de folios supera 180, aparece una alerta visual en la tarjeta "Estado del Protocolo".

📅 6. Calendario de Audiencias y Plazos

Cronograma de todas las actuaciones procesales programadas. Permite registrar audiencias vinculadas a expedientes específicos, con fecha, hora, cuantía y observaciones.

6.1 Selector de Fecha — Calendario dd-mm-aaaa

Todos los campos de fecha usan un calendario desplegable personalizado en lugar del selector nativo. El formato es siempre dd-mm-aaaa.

  • Haz clic en el campo de fecha → se despliega el calendario Fluent.
  • Navega meses con ‹ y ›. Clic en el día → queda marcado en azul.
  • Ajusta la hora directamente en el campo 08:00 dentro del mismo campo.
  • Hoy salta al día actual · Cancelar cierra sin guardar · Aceptar confirma.
  • Al editar una audiencia existente, la fecha se pre-carga automáticamente en el calendario.
💡 El campo muestra dd-mm-aaaa al usuario pero guarda el formato ISO internamente para compatibilidad con el backend.

Tabla de Audiencias

Columnas: Fecha y Hora, Expediente, Tipo de Audiencia, Juzgado, Observaciones y acciones.

  • 📋 Ver Ficha — Abre la Ficha Técnica completa de la audiencia (ver 6.2).
  • ✏️ Editar — Modifica los datos de la audiencia (requiere PIN).
  • 🗑️ Eliminar — Elimina la audiencia del cronograma (requiere PIN).

Botón: + Programar Audiencia

Abre el formulario de registro con los siguientes campos:

  • Tipo de Audiencia * — Selector con 9 tipos: Inicial, Preliminar, Juicio Oral y Público, Conciliación, Medidas Cautelares, Revisión o Apelación, Ejecución de Sentencia, Reconocimiento o Ratificación, Tutela o Protección.
  • ID del Expediente * — Código del expediente al que pertenece esta audiencia. Ejemplo: 2026-CIV-001. Debe coincidir con un expediente registrado.
  • Fecha y Hora * — Calendario desplegable personalizado. Muestra el formato dd-mm-aaaa. Ver sección 6.1.
  • 💰 Cuantía / Honorarios (L.) * — Monto en Lempiras. Se acumula en los "Honorarios Proyectados" del Dashboard.
  • Observaciones Especiales — Notas relevantes para esta audiencia. Ejemplo: Llevar peritos, preparar testigos.
  • Guardar en Cronograma — Registra la audiencia y la muestra en la tabla.
  • Cancelar — Cierra sin guardar.

6.2 Ficha Técnica de Audiencia

Documento interno de preparación para la audiencia. Se accede haciendo clic en 📋 Ver Ficha. Contiene:

  • Datos Generales del Expediente — Número de caso, materia/juzgado, partes del proceso y cuantía.
  • Anotaciones Pre-Audiencia (JURIS-AI) — Recordatorios automáticos generados por el sistema: verificar plazos fatales, confirmar comparecencia de peritos, revisar incidentes pendientes.
  • Espacio de Observaciones — Campo de texto editable para tomar notas en tiempo real durante la audiencia.
  • Resultado de la Audiencia — Selector con opciones: Pendiente / En Desarrollo, Favorable, Desfavorable, Suspendida / Reprogramada, Conciliada / Acuerdo Logrado.
  • Resumen de Resolución y Próximas Tareas — Campo de texto para registrar el resultado y las acciones a seguir.
  • 🖨️ Imprimir / PDF — Abre una ventana de impresión optimizada con el logo del despacho y formato profesional para llevar físicamente a la audiencia.
  • Guardar Ficha — Cierra el modal guardando las notas.

⚙️ 7. Configuración del Sistema

Panel de ajustes generales del despacho. Los cambios aquí se aplican a toda la aplicación (nombre en el encabezado, logo, seguridad y apariencia).

7.1 Identidad Institucional

  • Nombre del Despacho / Unidad — Aparece en el encabezado superior de la aplicación y en todos los documentos e informes generados.
  • Lema Legal (Opcional) — Subtítulo o eslogan del despacho. Se muestra debajo del nombre en el encabezado.
  • 📂 Seleccionar Archivo... (Logo)
    Abre el explorador de archivos para seleccionar el logo oficial del despacho. Se recomienda formato PNG de 200×200 píxeles. El logo se muestra en el menú lateral, en la Ficha Técnica de Audiencia y en los reportes PDF.

7.2 Catálogos Externos

  • Base de Incidentes Procesales (.JSON Externo)
    Campo que muestra la ruta del archivo JSON con el catálogo de incidentes procesales. Por defecto usa el catálogo integrado del sistema.
  • 📂 Enlazar Archivo...
    Permite vincular un archivo JSON personalizado con incidentes procesales adicionales. Útil para despachos con catálogos propios de tipos de incidencias.

7.3 Seguridad de Acceso — PIN de Administrador

  • PIN de Administrador — Contraseña numérica de 4 a 8 dígitos. Una vez configurado, el sistema solicitará este PIN antes de permitir editar o eliminar cualquier registro (clientes, expedientes, audiencias, instrumentos notariales).
⚠️ Si no configuras un PIN, cualquier usuario que acceda a la aplicación podrá modificar o eliminar datos sin restricción. Se recomienda siempre configurar un PIN.

7.4 Personalización Visual

  • Modo de Visualización
    Claro (Estilo Windows) — Fondo blanco, texto oscuro, ideal para uso en oficina con buena iluminación.
    Oscuro (Estilo Windows) — Fondo oscuro, texto claro, ideal para trabajo nocturno o en ambientes con poca luz.
    Personalizado — Habilita el selector de color de fondo libre.
  • Color de Acento — Los 5 círculos de colores cambian el color principal de botones, bordes y elementos destacados de la interfaz: Azul, Verde, Morado, Rojo y Dorado.

7.5 Escudo de Datos — Sistema de Respaldos

El sistema crea automáticamente un respaldo comprimido (.jgpbackup) de la base de datos y las plantillas cada vez que se cierra la aplicación. Los respaldos se guardan en la carpeta backups/ dentro del directorio de instalación.

  • 🛡️ Generar Respaldo Manual Ahora
    Crea un respaldo inmediato sin necesidad de cerrar la aplicación. Útil antes de realizar cambios importantes o al final de cada jornada laboral.

Botón Principal

  • 💾 Guardar Configuración — Aplica y persiste todos los cambios de esta pantalla. Obligatorio pulsar este botón para que el nombre del despacho, el lema y el PIN queden guardados.

7.6 Recuperación de PIN Olvidado

En caso de olvidar el PIN de Administrador, el sistema incluye un mecanismo de emergencia:

  • Vaya a la carpeta de instalación de JURIS-GESTIÓN-PRO.
  • Busque y ejecute el archivo llamado RECUPERAR_PIN.bat.
  • Siga las instrucciones en pantalla. Esto reseteará el PIN, permitiéndole entrar a Configuración y establecer uno nuevo.

⚖️ 8. JURIS-AI — Asistente Inteligente (Copilot)

Asistente de inteligencia artificial en formato panel overlay (estilo Copilot de Windows). No ocupa espacio permanente — se abre y cierra con el botón ⚖️ JURIS-AI en la barra superior.

Cómo Abrir y Cerrar

  • Pulsa ⚖️ JURIS-AI en la esquina superior derecha de la barra — el panel se desliza desde la derecha.
  • Para cerrar: pulsa el botón ✕ dentro del panel, o vuelve a pulsar el botón del header.
  • El área de trabajo principal queda accesible mientras el panel está abierto.

Cómo Usar

  1. Haz clic en una fila de la tabla de expedientes para establecer el "caso activo".
  2. Abre el panel JURIS-AI con el botón del header.
  3. Escribe tu consulta en el campo inferior del panel.
  4. Presiona Enter o el botón ➤ para enviar.
  5. JURIS-AI responde con el contexto del expediente activo (nombre del cliente, juzgado, materia).

Generación de Borradores

Cuando JURIS-AI genera un borrador completo, aparece el botón 📄 Abrir Documento Completo en Editor en el chat. Al pulsarlo, el borrador se carga directamente en el Editor de Documentos del expediente activo.

Ejemplos de Uso

  • "Redacta un escrito de contestación para el caso 2026-CIV-001"
  • "Genera un resumen ejecutivo de este expediente"
  • "¿Qué incidentes procesales son comunes en materia laboral?"
  • "Dame un borrador de poder especial para el cliente activo"
💡 Para que JURIS-AI tenga contexto específico del caso, primero haz clic sobre un expediente en la tabla. Sin caso activo, responde de forma genérica.
💡 JURIS-AI requiere una API Key en Configuración → api_key_ai para respuestas de IA completas. Sin ella, usa el catálogo integrado de plantillas e incidentes.

🛡️ 9. Barra Inferior — Estado del Escudo de Datos

Indicador permanente en la esquina inferior izquierda de la pantalla. Muestra el estado del sistema de respaldos automáticos.

  • ● Escudo de Datos OK — El sistema de respaldos está activo y funcionando correctamente. El punto verde indica que el último respaldo se completó sin errores.
💡 El respaldo se ejecuta automáticamente al cerrar la aplicación. El archivo generado tiene extensión .jgpbackup y contiene la base de datos completa más la configuración del sistema.

✅ 10. Flujo de Trabajo Recomendado (Paso a Paso)

Para un nuevo caso que llega al despacho, sigue este orden:

  1. Registrar al cliente → Ve a Clientes → + Nuevo Cliente → llena el formulario → ✅ Guardar.
  2. Abrir el expediente → Ve a Expedientes → + Nuevo Expediente → selecciona materia, cliente y juzgado → ✅ Guardar. El ID se genera solo.
  3. Programar la primera audiencia → Ve a Calendario → + Programar Audiencia → elige tipo, escribe el ID del expediente → haz clic en el campo de fecha → selecciona día en el calendario (dd-mm-aaaa) → ajusta hora → Aceptar → ingresa cuantía → Guardar.
  4. Redactar el primer escrito → En Expedientes, haz clic en la fila del caso → 📄 Redactar Documento → busca la plantilla adecuada → Cargar y Autocompletar → edita el texto → 📄 Exportar como Word (.docx).
  5. Actualizar el estado → Al avanzar el caso, haz clic en la fila → cambia a 🟡 En Proceso o 🔴 Cerrado según corresponda.
  6. Revisar el Dashboard → Ve a Dashboard para verificar que los honorarios y estadísticas se actualizaron correctamente.
💡 Al cerrar la aplicación, el Escudo de Datos crea automáticamente el respaldo. No es necesario ninguna acción adicional para proteger tu información.
